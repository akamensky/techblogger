<h1 class="post-title">How to fix WordPress database connection for MySQL 8</h1>
<span class="post-date">2018-10-27 | 0 comments</span>
<p>Very recently I have been upgrading software on my servers to latest one and took the risk of upgrading MySQL to latest 8.0.x version. You can read here how to <a href="https://www.akamenskiy.com/upgrade-wordpress-to-mysql-8/" title="upgrade WordPress to MySQL 8">upgrade WordPress to MySQL 8</a>. In this article I will show how to fix WordPress database connection for MySQL 8.</p>
<p>As many recently have experienced this resulted in my WordPress sites showing this error message:</p>
<blockquote><p>
Error establishing a database connection.
</p></blockquote>
<p>The cause of that is very simple though. <a href="https://dev.mysql.com/doc/refman/8.0/en/upgrading-from-previous-series.html#upgrade-caching-sha2-password" title="MySQL 8 have changed from using native encryption">MySQL 8 have changed from using native encryption</a> for passwords to a more standardized approaches. As a matter of fact, this change happened even earlier, in release 5.7. However, in MySQL 8 the team changed default value of <code>default_authentication_plugin</code> configuration variable from previous <code>mysql_native_password</code> to <code>caching_sha2_password</code>.</p>
<p>What this means is that when a client connects to MySQL without specifying which password encryption method it uses MySQL assumes it to be using <code>caching_sha2_password</code>. But most of clients currently in use are assuming to use <code>mysql_native_password</code>.</p>
<p>Here is how you can fix this issue:</p>
<p>Firstly you need make sure you have a backup of any files you will be changing. In this guide we will be changing MySQL configuration file, so let&#8217;s back it up first. I always recommend using current date and backup sequence number in backup names to easier identify them and being able to keep multiple backups from before:</p>
<pre><code class="language-shell line-numbers">$ sudo cp /etc/my.cnf /etc/my.cnf.backup.20181027.001
</code></pre>
<p>Now let&#8217;s go and edit MySQL configuration file:</p>
<pre><code class="language-shell line-numbers">$ sudo nano /etc/my.cnf
</code></pre>
<p>In this file locate <code>[mysqld]</code> section there and add the configuration lines as follows:</p>
<pre><code class="language-ini line-numbers">[mysqld]
default_authentication_plugin=mysql_native_password
</code></pre>
<p>There may be other configuration lines between those, but there should be no other section header between them.</p>
<p>After this you need to restart MySQL server process:</p>
<pre><code class="language-shell line-numbers">$ sudo systemctl restart mysqld
</code></pre>
<p>All done. Now your WordPress should be able to connect to MySQL 8 without any problems. In the future when the community will update <code>mysqlnd</code> client library to use new encryption methods this may be no longer needed, but as of now it still is.</p>
<p>Please let me know in the comments if this solution helped you to fix wordpress database connection for MySQL 8!</p>
